BOM-浏览器对象模型

不乱于心，不困于情。不畏将来，不念过往。如此，安好。 ——丰子恺《不宠无惊过一生》

Ziyi Xie

Apr 26, 2019 • 7 min read

前言

虽 ECMAScript 视 BOM 为 JavaScript 核心，但实际上 BOM 是使用 JavaScript 开发 Web 应用程序的核心。因 BOM 提供很多与页面无关却包含很多属性与方法的浏览器功能对象，从而与浏览器窗口交互。如生活中常见的刷新页面、滚动页面、放大缩小、历史条目...

BOM 不像 JavaScript 、 DOM 有 ECMA 、 W3C 制定标准，不同浏览器存在差异，兼容性较差。

正文：

window对象：

BOM 的核心是 window 对象，表示浏览器的实例。

window在浏览器中有双重身份：ECMAScript中的Global对象、浏览器窗口的JavaScript接口。

对比顶级对象：DOM=>document、BOM=>window。

  • Global作用域：
    通过var声明的所有全局变量和函数都会变成window对象的属性和方法。

  • 常见事件：
    页面加载事件 => 页面内容完全加载完成（脚本文件、CSS...）时触发，且传统注册事件方式仅一次，以最后出现为准。
    window.onload=function(){} == window.addEventListener("load",function(){})
    DOMContentLoaded快于load，因不考虑图片、flash、CSS等.
    调整窗口大小事件 => 窗口大小变化则触发执行处理函数。常应用于响应式布局，移动端项目缩小某一尺寸则隐藏某功能块。
    window.onresize=function(){} == window.addEventListener("resize",function(){})

  • 窗口关系：
    window.parent、window.top、window.self

  • 窗口位置与像素比：
    window对象位置可以通过不同属性与方法确定。现代浏览器提供screenLeft、screenTop属性表示窗口相对于屏幕左侧与顶部位置，返回CSS像素。

<script>
function myFunction() {
  var zsWindow = window.open("", "zsWin");
  zsWindow.document.write("<p>This is 'zsWin'");
  zsWindow.document.write("<br>ScreenLeft: " + zsWindow.screenLeft);
  zsWindow.document.write("<br>ScreenTop: " + zsWindow.screenTop + "</p>");
}
</script>

<button onclick="myFunction()">Open "zsWin"</button>

正常窗口浏览器:

缩放窗口浏览器:

移动窗口：moveTo()要移动位置的绝对坐标、moveBy()要移动位置的相对坐标。

  • 窗口大小：
    不同浏览器窗口大小不同。
    返回浏览器窗口自身大小：outerWidth、outerHeight
    返回浏览器窗口中的视口大小innerWidth、innerHeight

  • 视口位置：
    浏览器窗口尺寸通常无法完整显示整个界面，用户通过滚动在有限的 视口 中查看文档。
    文档相对视口滚动的距离属性：window.pageXoffset/window.scrollX和window.pageYoffset/window.scrollY
    滚动页面方法：scroll()、scrollTo()滚动到页面某值处、scrollBy()相对当前视口滚动

  • 导航与打开新窗口：
    window.open()方法用于导航到指定URL，也可用于打开新浏览器窗口。
    此方法接受4个参数：要加载的URL、目标窗口、特性字符串、布尔值。
    目标窗口可以为自定义或特殊窗口名（_self、_parent、_top、_blank）
    其中布尔值表示新窗口在浏览器历史记录中是否替代当前加载页面。

  • 定时器：
    setTimeout()指定一定时间后执行代码、setInterval()指定每隔一段时间执行代码。

  • 系统对话框：
    alert()警告框、confirm()确认框、prompt()提示框。

location：

location 对象提供当前窗口中加载文档的信息，以及常用的导航功能。location 对象既是 window 的属性，又是 document 的属性。即window.location与document.location指的是同一对象。

  • location 对象内容

假设浏览器当前加载的 URL 为 https://zairesinatra.com:90/xzy/?usr=zs#frontend

属性 值 说明
location.hash #frontend URL 散列值、没有则为空字符串
location.host https://zairesinatra.com:90 服务器名及端口号
location.hostname https://zairesinatra.com 服务器名
location.href https://zairesinatra.com:90/xzy/?usr=zs#frontend 完整的 URL
location.pathname /xzy/ 路径或文件名
location.port 90 端口号
location.protocal https: 协议
location.search ?usr=zs 查询字符串
location.origin https://www.zairesinatra URL 源地址、只读
  • 查询字符串

解析查询字符串、并返回一个以每个查询参数为属性的对象的函数封装

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>查询字符串</title>
</head>
<body>
    <span>getQueryStringArgs</span>
    <script>
        let getQueryStringArgs = function () {
            // 去?、取得没有问号开头的查询字符串
            // substring() 方法返回一个字符串在开始索引到结束索引之间的一个子集,或从开始索引直到字符串的末尾的一个子集
            let zy = (location.search.length > 0 ? location.search.substring(1) : ""), args = {};
            // 把每个参数保存在args字面量对象
            // split() 方法使用指定的分隔符字符串将一个 String 对象分割成子字符串数组
            for (let item of zy.split("&").map(haha => haha.split("="))) {
                let name = decodeURIComponent(item[0]), value = decodeURIComponent(item[1]);
                if (name.length) {
                    args[name] = value;
                }
            }
            return args
        }
        window.location.search="?usr=xzy&age=23"
        let args = getQueryStringArgs()
        console.log(args["usr"]); // xzy
        console.log(args["age"]); // 23
    </script>
</body>
</html>

现阶段通过 URLSearchParams 提供一组标准的 API 方法可检查、修改查询字符串

let str = "?usr=xzy&age=23"
let searchParams = new URLSearchParams(str);
console.log(searchParams.toString()); // usr=xzy&age=24
console.log(searchParams.has("usr")); // true
console.log(searchParams.get("age")); // 23
  • 操作地址

  1. 可通过修改 location 对象修改浏览器地址 => 最常见使用 assign() 方法传入 URL
    location.assign("www.zairesinatra.com") // 立即启动导航到新URL,同时浏览器历史记录新增一条记录

  2. 修改 location 对象属性也会修改当前加载的页面

navigator：

navigator 是客户端标识浏览器的标准、各浏览器属性不一

  • 检测插件

  • 处理注册程序

实际开发中用的最多莫属 navigator 对象中的 geolocation 对象。

screen：

window 的 screen 对象属性在编程中用的较少。纯粹是客户端显示器能力信息，如像素宽、像素高。了解即可。

history：

history 对象表示当前窗口首次使用以来用户的导航历史记录。 history 是 window 的属性且每个 window 都有专属的 history 。例 在 Chrome 中新建一个窗口，访问页面后仅留下本次操作的 history 对象。

  • 导航：
  1. 方法-window.history.go()
    定义: 接受一个整数参数，沿任意方向导航，表示前进或后退多少步
    示例:
go(1) // 前进一页
go(-1) // 后退一页

// 不同与window.history.go()中可用参数表示跳转历史条目,back()与forward()中参数无效
history.back() // 后退一页
history.forward() // 前进一页
  1. 属性-window.history.length
    定义: 记录历史条目数
    示例:
Google-a-b-c-d // window.history.length == 5
  • 历史状态管理：

目的: 为 HTML5 中 history 对象增加方便的状态管理特性
定义: 改变浏览器 URL 而不会加载页面: history.pushState()接受三个参数：state对象、新状态的标题、(可选的)相对URL。
示例:

let stateObject = {foo:zs}
history.pushState(stateObject, "zairesinatra", "zs.html")
// 完成时状态信息推入历史记录、地址栏改变、浏览器不会发送服务器请求
// pushState()会创建新的历史记录,启用"后退"按钮
