Vue双向数据绑定原理

前言：

前端三大框架之一的Vue。也在国内使用占比最大的用于构建用户界面的渐进式框架。

对Vue进化路线的认知：
从目前理论来说，最新的3.x版本还不推荐用于生产项目，周边生态还需要时间完善。那么本文主要通过2.x版本作为地基来阐述。

正文：

MVVM 即模型-视图-视图模型(Model-View-ViewModel)，是一个软件架构的设计模式。

MVVM的发展史：

最早的HTML页面是完全静态的网页，它们是预先编写好的存放在Web服务器上的html文件。就相当于在学习Node.js过程中通过Express第三方包托管静态资源。

app.use(express.static(path.join(__dirname, 'zspublic')));

但随着计算机的发展，成千上万用户的涌入，这时针对不同的页面需求与功能，不能仅仅通过向服务器get一份新的 html 文件这么简单。终于，能让浏览器自己修改 HTML 页面的 JavaScript 在95年运行了起来。但由于原生 API 不那么好用， jQuery 运营而生。在随着 MVC 模式(Model–view–controller)这种服务器配合让 JavaScript 修改服务器渲染后的数据的方式慢慢不能满足页面的交互性。MVVM 模式横空出世。

MVVM的理解：

MVVM的出现促进了 GUI(Graphical user interface图形用户界面) 。前后端业务逐渐分离，效率开始逐步提高。

MVVM的核心是 ViewModel 层。向上与视图层进行双向数据绑定，向下与 Model 层通过接口请求进行数据交互(Ajax<=>JSON)。

MVVM 的组成可以化分为以 HTML、CSS、模板文件为集合的 view 视图层，以 JavaScript、runtime、compiler 为集合的 ViewModel 视图模型层，和以业务处理逻辑 Node.js、Java...和数据库 MySQL、MongoDB...组成的 Model 层。

Vue双向数据绑定原理：

  • Vue 双向数据绑定是通过 数据劫持，并结合 发布-订阅模式 的方法来实现的，也就是说数据和视图同步互为变化条件与结果。

访问或者修改对象的某个属性时，除了执行基本的数据获取和修改操作以外，还基于数据的操作行为，以数据为基础去执行额外的操作。（搜索关键词高亮、联想、数据渲染...）

  • 核心API：Object.defineProperty() 方法以及基于数组的数据修改方法 push、pop、unshift、shift、slice、sort、reverse...

静态方法 Object.defineProperty(obj, prop, descriptor) 直接在对象上定义一个新的属性，或者修改一个对象上已有的属性，并返回该对象。

参数 说明
obj 定义属性的对象
prop (Symbol)定义或修改的属性的名称
descriptor 正在定义或修改的属性的描述符(方法函数)

概括就是通过此 API 来定义一个属性。当调用属性时使用 get 方法；给这个属性赋值时，使用 set 方法。

var zsobj = {};
Object.defineProperty(zsobj,'biubiubiu',{
  // value: 23,
  get: function () {
    console.log('调用新定义的 biubiubiu 属性的 get 方法')
  },
  set: function(args){
    console.log('调用新定义的 biubiubiu 属性的 set 方法, 传入的值是' + args);
  }
});
zsobj.biubiubiu; // => 调用新定义的 biubiubiu 属性的 get 方法
zsobj.biubiubiu = 'zairesinatra' // 调用新定义的 biubiubiu 属性的 set 方法, 传入的值是zairesinatra

var zy = {};
Value = 23;
Object.defineProperty(zy, 'one', {
  writable: true,
});
// 不能尝试混合两者: 数据描述符、访问器描述符
Object.defineProperty(zy, 'onee', {
  get () { return console.log(Value) },
  set (newValue) { Value = newValue; },
  enumerable: true,
  configurable: true
});
zy.onee // 23
zy.onee = '24'
zy.onee // 24

发布-订阅模式 和观察者模式（subject 与 observers 间存在一对多关系。当一个对象被修改时，则会自动通知依赖它的对象。观察者模式属于行为型模式。）很像，但是不同的是，观察者模式只有两个角色，且 obsevers 是知道 subject 的事件，但是在 发布-订阅 模式中，publisher、subscriber 不直接取得联系，通过类似于中间件（event bus）的帮助下代理事件通信，传入消息并完成相应的分发。更多点此

实现双向绑定：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>mvvm</title>
</head>
<body>
    <input type="text">
    <span></span>
    <script>
        // 获取文档中的第一个DOM元素
        let input = document.querySelector('input');
        let span = document.querySelector('span');
        // 建立 event channel
        let obj = {};
        let value = '';
        Object.defineProperty(obj,'dataDescribe',{
            configurable: true | false, // 如果是false则不可以删除
            enumerable: true | false, // 如果为false则在枚举时候会忽略
            writable: true | false // 如果为false则不可采用数据运算符进行赋值
        });
        Object.defineProperty(obj, 'zsget', {
            set: function (newValue) {
                // fire event
                value = newValue
                input.value = newValue
                span.innerText = newValue
            },
            get: function () {
                return value
            }
        })
        input.addEventListener('keyup',(e) => {
            obj.zsget = e.target.value
        })
        input.value = obj.zsget
        span.innerText = obj.zsget
    </script>
</body>
</html>

结束

不论框架怎么改变，还是稳固的 JavaScript 作为底层实现。
