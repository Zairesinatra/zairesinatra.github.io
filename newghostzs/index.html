Ghost更新与迁移

Ubuntu 最近 Ghost 以及 Node 版本更新过程小结。

Ziyi Xie

Jul 23, 2021 • 6 min read

由于 Githubpage 的静态编译 Ghost 在表现上不同于本站的反向代理，对部分功能与效果支持不好。为同步需求，记录本次的 TX云 Ubuntu 中 Ghost 与 Node 版本更新过程以及 Nginx 再配置。（又回来了）

Node

最新的 Ghost 版本需要 Node.js 14.x ，故安装适用 Ubuntu 的 二进制发行版。

# Using Ubuntu 添加 Node.js ppa(Person Package achives) 安装 Node.js
# cURL is an abbreviation for Client URL Request Library which is used to transfer data from one place to another place. 
# -f (--fail) 表示在服务器错误时, 阻止一个返回表示错误原因的 html 页面
# -S (--show-error) 使curl在失败时显示错误消息 -s (--silent) 不显示进度表或错误信息的安静模式
# -L (--location) 如果服务器报告请求的页面已移动到其他位置,将使curl在新位置上重新执行请求
curl -fsSL https://deb.nodesource.com/setup_14.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v
# v14.17.2

解决方法是给 ghost 文件夹当前用户权限：

sudo chown /var/www/ghost

这时由于此前更新准备时出现的报错，从 Ghost-CLI 1.5.0 开始，root不允许运行 Ghost 命令。因此，需要将当前安装迁移到新的 Linux 用户。

We discovered that you are using the DigitalOcean One-Click install. You need to create a user with regular account privileges and migrate your installation to this user.

添加一个新用户，并将用户添加到 sudo 组

$ adduser <user>
$ usermod -aG sudo <user>

创建一个新的实例文件并移动到新的用户位置

mkdir -p /home/<user>/.ghost/

chown <user>:<user> /home/<user>/.ghost

mv /root/.ghost/config /home/<user>/.ghost/config

chown <user>:<user> /home/<user>/.ghost/config

接下来，切换到 Ghost 安装目录（/var/www/ghost/在 DigitOcean One-Click 安装中是默认的）以更改安装文件的所有权：

$ cd /var/www/ghost
$ find . -group root -user root -exec chown <user>:<user> {} \;

以新用户身份登录：

$ su - <user>

完成后则可继续进行 ghost 更新

Nginx

而此时不能通过 www.zairesinatra.com 访问网站而仅能使用 zairesinatra.com 访问。那么有两种方法，第一种是配置 Nginx 使 www.zairesinatra.com 同样渲染 Ghost；第二种是使得 www.zairesinatra.com 进入则重定向到 zairesinatra.com 。

出于 SEO 的目的，Ghost 本身只能指向一个域。接下来详述第二种方法执行过程。

# Determine your secondary URL
ghost config url https://my-second-domain.com

# Get Ghost-CLI to generate an SSL setup for you:
ghost setup nginx ssl

# Change your config back to your canonical domain
ghost config url https://my-canonical-domain.com

# Edit the nginx config files for your second domain to redirect to your canonical domain. In both files replace the content of the first location block with:
return 301 https://my-canonical-domain.com$request_uri;

# Get nginx to verify your config
sudo nginx -t

# Reload nginx with your new config
sudo nginx -s reload

这里在 sudo nginx -t 时，可能会遇见如下 Error。

newghostzs@VM-4-10-ubuntu:/etc/nginx/sites-available$ sudo nginx -t
nginx: [warn] conflicting server name "www.zairesinatra.com" on 0.0.0.0:80, ignored
nginx: [warn] conflicting server name "www.zairesinatra.com" on [::]:80, ignored
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful

这里的警告意思是重复绑定了 server name，在 /etc/nginx/sites-enabled 文件夹中搜索域：

$ grep -r mydomain.com /etc/nginx/sites-enabled

在端口 80 的多个服务器块中看到域名，则需要删除重复的域名，然后重新启动 Nginx。一般将配置文件单独分离出来容易出现这个错误。
