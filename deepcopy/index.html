Javascript深拷贝与浅拷贝

Adversity and loss make a man wise.

Ziyi Xie

Jan 1, 2020 • 3 min read

开始

记录一下Javascript中的重点-深、浅拷贝。实在是居家旅行，面试加薪，必备良药。

正文

必备基础

Javascript数据类型：基本数据类型、复杂数据类型
基本数据类型：null,undefined,string,number,boolean,symbol
复杂数据类型（引用类型）：Object,Function,Array,...

基本数据类型数据存储于栈、复杂数据类型数据存储于堆
因此基础数据类型都是按值访问、数据在栈内存中的存储与使用方式类似于数据结构中的堆栈数据结构，遵循后进先出的原则。
当访问堆内存中的引用数据类型时，实际上首先是从栈中获取了该对象的地址引用（或者地址指针），然后再从堆内存中取得我们需要的数据。

抛出问题

浅拷贝: 仅是复制了引用（地址），换句话说，复制了之后，原来的变量和新的变量指向同一个东西，彼此之间的操作会互相影响。

var obj = {
    age:23,
    name:'zs',
    // 对象在内存中新开辟空间存放
    hobby:{
        code:'js'
    }
}
var g = {};

// 浅拷贝
for(var i in obj){
    // i是属性名而obj[i]是属性值
    g[i]=obj[i];
}
// 这里拷贝的复杂对象仅仅是拷贝的地址
console.log(g);
// 此时如果修改了g内数据也会影响obj内数据
g.hobby.code = 'C++';
console.log(obj);
//实现浅拷贝ES6语法糖-> Object.assign(target,source)

解决思路

因浅拷贝仅拷贝一层，更深层次对象级别只拷贝引用。为了避免互相影响，深拷贝需要拷贝每一层，即拷贝所有的属性,且地址也与原来的不同,当改变属性时就也不会影响原来的数据。

  • 简单数据直接赋值、复杂数据进入遍历
  • 函数递归完成

深拷贝代码

var obj = {
            age:23,
            name:'zs',
            // 对象在内存中新开辟空间存放
            hobby:{
                code:'js'
            },
            skills:['calligraphy','guitar']
        }
var g = {};
function foo (){}
// 深拷贝
// 传递两个参数:要拷贝谁，拷贝给谁
function deepCopy(newobj,oldobj){
    for(var i in oldobj){
        // 获取属性值
        var hhh = oldobj[i];
        // 判断数据属于那种数据类型
        // 属于数组
        if(hhh instanceof Array){
            // 新对象属性是数组
            newobj[i]=[];
            // deepCopy(newobj[i],oldobj[i])
            deepCopy(newobj[i],hhh)
        }
        // 属于对象
        else if(hhh instanceof Object){
            newobj[i]={};
            // 把值给属性
            deepCopy(newobj[i],oldobj[i])
        }
        // 属于简单数据类型
        else {
            newobj[i] = oldobj[i]
        }
    }
}
deepCopy(g,obj);
console.log(g)
// 注意函数instanceof属于Object
console.log(foo instanceof Object)
// 同时数组也属于Object,故先后顺序Array在Object之前,否则数组就被对象占位了
console.log([1,2,3] instanceof Object)
// 这里修改g则不会对原obj进行修改
g.hobby.code = "C++";
console.log(obj)
