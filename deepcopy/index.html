堆、栈与深、浅拷贝

Adversity and loss make a man wise.

Ziyi Xie

Jan 1, 2020 • 5 min read

栈与堆

栈是内存中一片连续用于存储 局部变量 和 函数参数 的线性结构。严格先进后出，且入栈和出栈的操作仅是栈指针在内存地址中的上下移动。栈在函数调用时创建，调用结束则消失。

堆是内存中一大块独立存储空间，在栈中存储不了的数据（对象），就会被存储在堆中，而栈中仅存对该数据的引用（也就是该块数据的首地址）。为避免内存溢出，在 C 中堆内存是需要手动清理的，而在 JAVA 之流存在 Garbage Collection 机制协助清理堆中已不被使用的数据。

function Closure () {
    let num = 0;
    return function () {
        num++;
        return num;
    }
}
let Closuree = Closure();
Closuree(); // 1
Closuree(); // 2

观察这段闭包代码是否让你顿时怀疑人生，因为上述栈的描述里，在调用完函数后，栈会销毁，那么就不应该存在后续的 num 仍然会发生改变。实际上在本例中的 num 变量并没有存储在栈中，而是在堆内存里，用一个特殊的对象 [[Scopes]] 保存。

在 JavaScript 中，变量分为三种类型：局部变量、被捕获变量、全局变量。其中 被捕获变量 较为少见——在函数中声明，却在函数返回后仍有未执行作用域（函数或是类）使用到该变量，那么该变量就是被捕获变量（可以简单理解为闭包中返回带有的参数变量）。

[[Scopes]] 是 Chrome 开发人员工具在内部添加和使用的私有属性，在源代码中。它显示函数范围内的变量，即可以从该函数访问哪些变量。

总结：只有局部变量的基本数据类型的值是存放在栈中的，其他（即其他的基本数据类型和所有的引用类型）都是地址存放在栈中，值存放在堆中。

深、浅拷贝

从上述可以得知，深、浅拷贝是对于引用类型来说的，引用类型的值实际存储在堆 Heap 内存中，在栈 Stack 中只存储堆内地址的引用，而这个地址的引用指向堆内存中的值。

浅拷贝仅拷贝一层，对更深层次对象级别只拷贝引用，那么在操作数据时难免出现对象互相影响的情况。深拷贝需要拷贝每一层的所有的属性，且地址也与原来的不同，那么改变属性时也就不会影响原来的数据。

实现深拷贝的三种方式：

  • lodash 实用程序库
$ npm i -g npm
$ npm i --save lodash

// 加载完整的构建
var _ = require('lodash');
let obj = [
    {
        a: 1,
        b: 2
    },
    {
        a: 1,
        b: 2
    }
]
let objj = _.cloneDeep(obj)
console.log('相等比较 ==>', obj === objj) // 相等比较 ==> false
objj.c = 3
console.log('obj ==>', obj); // obj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 } ]
console.log('objj ==>', objj); // objj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 }, c: 3 ]
obj[0].c = 3;
console.log('obj ==>', obj); // obj ==> [ { a: 1, b: 2, c: 3 }, { a: 1, b: 2 } ]
console.log('objj ==>', objj); // objj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 }, c: 3 ]
  • JSON.parse() 和 JSON.stringify() 组合
// 只限于处理可被 JSON.stringify() 编码的值: Boolean, Number, String, Object, Array
let obj = [
    {
        a: 1,
        b: 2
    },
    {
        a: 1,
        b: 2
    }
]
let objj = JSON.parse(JSON.stringify(obj));
console.log('相等比较==>', obj === objj) // 相等比较==> false
objj.c = 3
console.log('obj ==>', obj); // obj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 } ]
console.log('objj ==>', objj); // objj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 }, c: 3 ]
obj[0].c = 3;
console.log('obj ==>', obj); // obj ==> [ { a: 1, b: 2, c: 3 }, { a: 1, b: 2 } ]
console.log('objj ==>', objj); // objj ==> [ { a: 1, b: 2 }, { a: 1, b: 2 }, c: 3 ]
  • 自行造轮子
// 深拷贝代码
// 思路: 简单数据直接赋值、复杂数据进入遍历 + 递归
var obj = {
    age:23,
    name:'zs',
    // 对象在内存中新开辟空间存放
    hobby:{ code:'js' },
    skills:['calligraphy','guitar']
}
var g = {};
function foo (){}

// 传递两个参数:要拷贝谁，拷贝给谁
function deepCopy(newobj,oldobj){
    for(var i in oldobj){
        // 获取属性值
        var hhh = oldobj[i];
        // 判断数据属于那种数据类型
        // 属于数组
        if(hhh instanceof Array){
            // 新对象属性是数组
            newobj[i]=[];
            // deepCopy(newobj[i],oldobj[i])
            deepCopy(newobj[i],hhh)
        }
        // 属于对象
        else if(hhh instanceof Object){
            newobj[i]={};
            // 把值给属性
            deepCopy(newobj[i],oldobj[i])
        }
        // 属于简单数据类型
        else {
            newobj[i] = oldobj[i]
        }
    }
}
deepCopy(g,obj);
g.hobby.code = "C++";
console.log(g); // { age: 23, name: 'zs', hobby: { code: 'js' }, skills: [ 'calligraphy', 'guitar' ] }
console.log(obj); // { age: 23, name: 'zs', hobby: { code: 'C++' }, skills: [ 'calligraphy', 'guitar' ] }

浅拷贝: 仅是复制了引用（地址），换句话说，复制了之后，原来的变量和新的变量指向同一个东西，彼此之间的操作会互相影响。

实现深拷贝的两种方式：

  • ES6语法糖-> Object.assign(target,source)
var obj = {
    age:23,
    name:'zs',
    // 对象在内存中新开辟空间存放
    hobby:{
        code:'js'
    }
}

// 浅拷贝
var objj = Object.assign({},obj)
// 这里拷贝的复杂对象仅仅是拷贝的地址
console.log(objj); // { age: 23, name: 'zs', hobby: { code: 'js' } }
// 此时如果修改了g内数据也会影响obj内数据
objj.hobby.code = 'C++';
console.log(obj); // { age: 23, name: 'zs', hobby: { code: 'C++' } }
  • 自行造轮子
var obj = {
    age:23,
    name:'zs',
    // 对象在内存中新开辟空间存放
    hobby:{
        code:'js'
    }
}
var g = {};

// 浅拷贝
for(var i in obj){
    // i是属性名而obj[i]是属性值
    g[i]=obj[i];
}
// 这里拷贝的复杂对象仅仅是拷贝的地址
console.log(g); // { age: 23, name: 'zs', hobby: { code: 'js' } }
// 此时如果修改了g内数据也会影响obj内数据
g.hobby.code = 'C++';
console.log(obj); // { age: 23, name: 'zs', hobby: { code: 'C++' } }
