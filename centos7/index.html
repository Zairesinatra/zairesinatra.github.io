Ziyi Xie

Jul 16, 2019 • 5 min read

CentOS 即 "RHEL (Red Hat Enterprise Linux)" 发行版本，每个版本都会获得七年安全更新的支持。而 Ubuntu 是从 Debian 开发版本衍生而来的。在包管理上，CentOS 采用 RPM 与 YUM 的集成。

Mac 连接

Centos 远程服务器在重装系统后会产生导致 Mac 无法正常连接的情况，伴随出现 "REMOTE HOST IDENTIFICATION HAS CHANGED" 的终端提示。此时需要删除 Mac 保存的配置文件才能重新连接。

出现 "Permission denied (publickey,gssapi-keyex,gssapi-with-mic)." 的拒绝，则需要在服务器中的 sshd_config 更改设置，最后重启服务即可。

# centOS
vim /etc/ssh/sshd_config # 更改为 PasswordAuthentication yes

Nginx

Nginx 相关配置文件位于 /etc/nginx/ 目录，服务配置文件都以 .conf 结尾存储在 /etc/nginx/conf.d 目录。若域名是 nginxdevtest.com，那么对应的配置文件应该为 /etc/nginx/conf.d/nginxdevtest.com.port.conf。

# 将要部署的代码文件一般推荐存放位
/home/<user_name>/<site_name>
/var/www/<site_name>
/var/www/html/<site_name>
/opt/<site_name>
/usr/share/nginx/html
# Nginx 日志文件(access.log 和 error.log)
/var/log/nginx/
# 复用配置代码文件
/etc/nginx/snippets
# 相关的配置文件
/etc/nginx/
# 主配置文件
/etc/nginx/nginx.conf

# 安装 Nginx
sudo yum install nginx
# 设置开机启动|关闭
sudo systemctl enable|disable nginx # 运行此命令以后,会输出软连接关联信息
# 启动关闭重启 —— systemctl - Control the systemd system and service manager
sudo systemctl start|stop|restart nginx
# 查看运行状态
sudo systemctl status nginx
# 查看80口是否被占用
sudo lsof -i :80 # 查看80的nginx -> sudo lsof -i :80 | grep nginx
# 杀死端口
lsof -P | grep ':80' | awk '{print $2}' | xargs kill

Tomcat

  • 安装配置

详见本人 tomcat 博文。

  • tomcat 启动成功外网无法访问

部署的服务器没有将处于过滤状态的 8080 端口放通，所以 tomcat 无法从外部进行访问。需从控制台进入云服务器对防火墙规则进行配置。局域网没有固定的公网 IP，所以不能将局域网的 IP (192.168.124.2) 设置为安全组。被 NAT 转换的公网 IP 可以通过国内或国外的网站进行查询检测。

防火墙

CentOS7 默认使用 firewall 作为防火墙，而 CentOS6 及以下使用 iptables。

  • firewall
# 查看状态
systemctl status firewalld # 或 firewall-cmd --state
# 启动|禁用防火墙
systemctl start|stop firewalld
# 重启防火墙
firewall-cmd --reload
# 设置开机启动|停止并禁用开机启动
systemctl enable|disable firewalld
...
  • iptables
# 开启防火墙(重启后永久生效)
chkconfig iptables on
# 关闭防火墙(重启后永久生效)
chkconfig iptables off
# 开启防火墙(即时生效，重启后失效)
service iptables start
# 关闭防火墙(即时生效，重启后失效)
service iptables stop
# 重启防火墙
service iptables restartd
