BUG: 非顶级域名不可进入网页

顶级域名访问页面成功，使用二级域名却出现访问失败、受限或Nginx页面的Welcome Page的解决方案。

Ziyi Xie

Dec 16, 2019 • 3 min read

Bug出现:

顶级域名访问页面成功，使用二级域名却出现访问失败、不安全导致的受限或仅显示Nginx页面的Welcome Page。

储备:

域名是由一串用点分隔的名字组成的Internet上某一台计算机或计算机组的名称。用于在数据传输时对计算机的定位标识。

BUG排查:

  • 域名解析;
  • 配置.conf文件;
  • 缓存存在;

BUG解决:

在腾讯云的 域名解析列表 检查是否配置了解析。同时确保 主机记录 、 记录类型 、记录值 的正确。

  • 通过排查定位于conf.d文件配置有误，无关解析、CLB之流...

  • 登陆云服务器通过查询 Nginx 位置寻找配置文件nginx.conf：
    whereis nginx

  • 若配置无误则深层排查virtual host configs

  • 进入 顶级域名.conf进行排查

  • 发现server_name少配置一项二级域名，二级域名通过空格隔离补在顶级域名之后。

  • 重启 Nginx：
    service nginx restart

  • 此时问题解决，若仍显示报错尝试移动端(手机)登陆，若移动端登陆成功则清除浏览器缓存。

注意事项:

  • 若配置 域名.conf 权限不够，则尝试切换用户
    sudo vim 域名.conf
  • 实际操作中查询过ghost配置文件

记录一下自己的 ssl 配置：

首先从腾讯云申请一个免费的 ssl 证书，过了审核后下载下来。因本人是使用的 Nginx 。所以解压证书文件夹所需要的主要的文件就是其中 Nginx 文件夹中的：xxx.rt证书文件、xxx.key秘钥文件。通过自己喜欢的方式上传到云服务器。我这边使用的是 Gitee 。
归类：资源类文件=>/var/www、执行软件与包内容=>/etc/nginx
将ssl放在 /etc/nginx 目录下，进入/etc/nginx/sites-enabled配置.conf。这里因为我又传了一个导航页面上去。所以得重新配置一下。

注意这里如果使用 mv A B要查看是否有存在目录的名称，若没有，则是更改文件名，若有则是移动文件。

配置完通过以下代码查看一下:以递归方式查找文件里所有符合条件的字符串并标记行号

grep -rn nav *

后续新追加传上去的的导航页面也确实帮我节约了挺多本机书签有效空间。
