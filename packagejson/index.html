package-lock.json与package.json

"If life were predictable it would cease to be life, and be without flavor." -Eleanor Roosevelt

Ziyi Xie

Jun 1, 2020 • 2 min read

前言

做demo引入jquery时发现除了初始化产生的package.json文件，多了一个package-lock.json文件。这里来记录一下这个der到底是干嘛的。

正文

产生：

创建项目目录并初始化：
npm init -y => package.json
安装jQuery：
npm install jquery -s => package-lock.json

两者对比如图

区分：

在版本 5 中，npm 引入了 package-lock.json 文件。该文件旨在跟踪被安装的每个软件包的确切版本，以便产品可以以相同的方式被 100％ 复制（即使软件包的维护者更新了软件包）。
而这也是 package.json 一直尚未解决的特殊问题。 在 package.json 中，可以使用 semver 表示法设置要升级到的版本（补丁版本或次版本），例如：

  • 如果写入的是 〜0.6.0，则只更新补丁版本：即 0.6.1 可以，但 0.7.0 不可以。
  • 如果写入的是 ^0.6.0，则要更新补丁版本和次版本：即 0.6.1、0.7.0、依此类推。
  • 如果写入的是 0.6.0，则始终使用确切的版本。

优势：

  • 在我们工作或者学习时一般不会将 node-modules 文件夹提交 Git。那么接手项目的其他人可能使用 npm install复制项目，这里可能导致原始项目和初始化项目出现区别。因为可能存在补丁版或者次版本缺陷。这里可以打个比方提一句：在微信小程序开发的时候，使用的wepy1.x.x和wepy2.x.x是有区别的。
  • 那么 package-lock.json 可以固话当前安装的每个软件包版本。再运行 npm install时，npm 会使用确切的版本。
  • 同时，如果运行 npm update 时，package-lock.json 文件的依赖版本会被更新。

解释：

到了这里，可以解释一下这个 package-lock.json 文件中的几个字符含义了。

  • version：版本不多说了
  • resolved：指向软件包的位置
  • integrity：中文其实是"正直"的意思，这里表示用于校验软件包。
