清除方法

/* 样式模块 */
.parent{ width: 400px;border: 1px solid #424242; }
.left{ width: 200px;height: 200px;background: #00B26A;float: left; }
.leftt{ width: 120px;height: 120px;background: #06B4FE;float: right; }
.footer{ width: 900px;height: 100px;background: darkslateblue; }

<!-- 模板模块 -->
<div class="parent">
    <div class="left">left</div>
    <div class="leftt">leftt</div>
</div>
  • 额外标签

给需要清除浮动的元素后额外添加一个空标签设置 clear 属性。该属性指定一个元素是否必须移动（清除浮动后）到之前的浮动元素下方，适用于浮动和非浮动元素。在通俗的同时，却额外加入无意义的标签。

.clear{ clear:both; }

<div class="parent">
    ...
    <div class="clear">额外标签</div>
</div>
  • 父级 overflow

overflow 属性定义了当一个元素的内容太大而无法适应块级格式化上下文的处理方式。即通过触发 BFC 实现清除浮动。虽然代码简洁，但是无法解决显示要溢出元素的问题。

/* 修改parent样式;此处overflow必须为hidden */
.parent{ width: 400px;border: 1px solid #424242;overflow: hidden; }

额外提一点，overflow 设置为 hidden 主要作用于溢出隐藏、清除浮动、外边距塌陷等问题。外边距塌陷也称为外边距合并，是指两个在正常流中相邻（兄弟或父子关系）的块级元素外边距，组合在一起变成单个外边距，不过只有上下外边距才会有塌陷，左右外边距不会出现这种问题。

.big{ width: 200px;height: 200px;background: #00B26A;overflow: hidden; }
.small{ width: 120px;height: 120px;background: #35393b;margin-top: 30px; }

<div class="big">
    <div class="small"></div>
</div>
  • after 伪元素

给容器增加伪元素清除浮动样式的语义化较好，符合闭合浮动思想。但是存在兼容性问题，使用 zoom:1 触发 hasLayout。

/* 伪元素是行内元素 */
.clearfix:after{ display: block;clear:both;visibility: hidden;content: ""; }
.clearfix{ *zoom: 1; }
  • 双伪元素
/* display符合BFC即可:block、flex、table */
.clearfix:after,.clearfix:before{ content: ""; display: flex; }
.clearfix:after{ clear: both; }
.clearfix{ *zoom: 1; }

BFC

块格式化上下文 (Block Formatting Context，BFC) 是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域（自主规则，无关外部）。

布局规则

内部 Box 在垂直方向逐个排列；Box 垂直方向的距离由 margin 决定；同属一个 BFC 的两个相邻 Box 的 margin 会发生重叠；即使存在浮动，每个元素的 margin box 的左边，与包含块 border box 的左边相接触(对于从左往右的格式化，否则相反)；BFC 区域不会与 float box 重叠；BFC 是页面上的隔离独立容器；浮动元素的高度参与 BFC 的高度计算。

触发条件

根元素 <html>；float 值非 none；overflow 值非 visible；display 取值 table、grid、flex，inline-flex、table-cell、table-caption、inline-block；position 取值 absolute、fixed。

作用场景

  • 清除浮动
  • 自适应两栏布局
  • 阻止外边距塌陷
