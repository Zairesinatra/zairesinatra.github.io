微信WePY报错汇总

wepy 框架遇见的 BUGS 解决方案

Ziyi Xie

May 1, 2020 • 2 min read

从横空出世，到现在框架的百花齐放。微信的原生语法依然还是本人心头最喜欢的，因为很少产生一些稀奇古怪的错误。至于框架，Rax/Taro/uni-app/wepy...选择性还是很多，本文就 wepy 使用中遇见的一些问题记录相应的解决方案。

前景提要：
WePY@2.x.x 较 v1.x.x 变化较大，本文 WePY 版本为 version1.7.3。
v1.x.x与v2.x.x相比：node—modules文件夹一致，而前者的dist文件夹在后者中更新为weapp，即存放实时编译后代码。

  • WePY 初始化报错：
$ wepy init standard project_name
# 网络状态异常
downloading template   wepy-cli · Failed to download repo standard: read ECONNRESET

解决方案： 1.更换网络、科学上网 2.使用本地下载的初始化 template：进入 WePYGithub，选择下载需要的分支至本地，最后初始化项目由 wepy init standard project_name 修改为 wepy init 下载文件中 standard 存放路径 project_name

  • ESLint 语法报错：
.../src/app.wpy
15:1  error  More than 1 blank line not allowed  no-multiple-empty-lines

✖ 1 problem (1 error, 0 warnings)

.../src/pages/index.wpy
91:1  error  More than 1 blank line not allowed  no-multiple-empty-lines

✖ 1 problem (1 error, 0 warnings)

解决方案： 在对应报错目录中删除多余空行

  • 未找到入口 app.json 文件、或文件读取失败：
[ app.json 文件内容错误] app.json: app.json 未找到

解决方案： 补全项目根目录下的 project.config.json 文件缺少的代码："miniprogramRoot": "/dist"。这里注意路径的正确补全。

  • VSCode 中配置 Vetur后依旧 .wpy 文件报错：

解决方案： 在 VSCode 中 Settings 内关闭勾选的 Validation Script、Validation Style、Validation Template

  • MiniProgramError module "wxs/test.wxs.js" is not defined：

解决方案： 这里把报错的 test.wxs 后缀加上 .js 可以消除报错。产生的主要原因是缓存问题，待不报错后修改回即可。

  • 运行时未定义：
ReferenceError: regeneratorRuntime is not defined

解决方案： 1.将新运行时文件引入报错的文件中，这里注意在模块内 regeneratorRuntime 是以 runtime 运作，且此方法存在重新编译失效的问题。2.增强编译且更换版本可以较为简便的解决此问题，但是也存在切换版本后错误的复现。
