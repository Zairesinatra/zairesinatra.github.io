iterm2

  • 安装
brew install --cask iterm2
  • 调整

更改默认的黑白色调为彩色显示：iTerm2 -> Preferences -> Profiles -> Terminal -> Report Terminal Type -> xterm-256color

更改颜色主题：iTerm2 -> Preferences -> Profiles -> Colors -> Color Presets -> Pastel(Dark Background)

oh-my-zsh

  • zsh
# 通过 homebrew 安裝的 zsh 位置在 /usr/local/bin/zsh,而系统自带的路径在 /bin/zsh
$ which zsh
/bin/zsh
  • 安装
# curl
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
# wget
sh -c "$(wget -O- https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
# fetch
sh -c "$(fetch -o - https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

安装完成出现 ~/.oh-my-zsh 文件夹，且原存储在 .zshrc 中的环境变量会被迁移到 .zshrc.pre-oh-my-zsh 中，复制环境变量到新 .zshrc 即可。

Powerlevel10k

  • 安装
# kxsw
git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ~/powerlevel10k
echo 'source ~/powerlevel10k/powerlevel10k.zsh-theme' >>~/.zshrc
# 大陆用户
git clone --depth=1 https://gitee.com/romkatv/powerlevel10k.git ~/powerlevel10k
echo 'source ~/powerlevel10k/powerlevel10k.zsh-theme' >>~/.zshrc
  • 更改默认 THEME
$ vi ~/.zshrc

# 更改 ZSH_THEME
ZSH_THEME="powerlevel10k/powerlevel10k"
  • Homebrew
# 推荐homebrw一劳永逸免去更改ZSH_THEME的操作
brew install romkatv/powerlevel10k/powerlevel10k
echo "source $(brew --prefix)/opt/powerlevel10k/powerlevel10k.zsh-theme" >>~/.zshrc

自动提示与语法高亮

# 自动提示
$ git clone git://github.com/zsh-users/zsh-autosuggestions $ZSH_CUSTOM/plugins/zsh-autosuggestions
$ vim  ~/.zshrc # 将 plugins=(git) => plugins=(zsh-autosuggestions git)
# 语法高亮
$ brew install zsh-syntax-highlighting
$ source  /usr/local/Cellar/zsh-syntax-highlighting/0.7.1/share/zsh-syntax-highlighting/zsh-syntax-highlighting.zsh
$ source ~/.zshrc

操作快捷

# 新建窗口
command + t
# 垂直分屏
command + d
# 水平分屏
command + shift + d
# 分屏切换
command + ] | command + [
# 切换到指定位置的分屏
command + alt + 方向键
# 切换标签页
command + 数字
# 按方向切换标签页
command + 方向键
# 保存当前窗口快照
shift + command + s
# 快照回放
command + alt + b
# 重设p10k风格
p10k configure

Bugs 解决

  • 开启终端出现提示 => insecure directories
zsh compinit: insecure directories, run compaudit for list. Ignore insecure directories and continue [y] or abort compinit [n]?

解决方式是通过列出不安全的目录后，将对应的目录赋与权限。

$ compaudit # 列出不安全目录
/usr/local/share/zsh
$ cd /usr/local/share/
$ sudo chmod -R 755 ./zsh # 对应的目录赋与权限
