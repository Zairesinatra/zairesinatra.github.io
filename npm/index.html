NPM&YARN

来世不可待，往世不可追也 ——庄子 《人世间》

Ziyi Xie

Apr 25, 2020 • 4 min read

现开发中常见的主流包管理模块有 npm、yarn、bower。包管理模块，解决代码包的下载管理与关系依赖，允许用户从服务器下载第三方包和命令行程序到本地使用，甚至开源个人代码。

认知 NPM 通常从 CLI Command 及模块描述文件 package.json 开始，随后深入 npmrc、npm registry、semver、npm access 之流。

NPM

NPM CLI

CLI Commond Line Interface 对应 GUI 图形化介面，前者通常只支持键盘录入指令，不支持鼠标操作。Windows 的 PowerShell 或是 Mac 的 terminal 都是 CLI 工具。

# ---常用命令---
# 安装包的名称和版本号存到生产环境 dependencies 中
npm i _module_ -S
npm install _module_ --save
# 安装包的名称和版本号存到开发环境 devDependencies 中
npm i _module_ -D
npm install _module_ --save-dev
# 全局安装
npm i _module_ -g
# 清除npm缓存
npm cache clean -f
# 查看某个 package 的最新版本|在 npm 服务器上所有发布过的版本
npm view <packageName> version|versions # 具体版本
npm info <packageName> # 更加详细
# npm 查看本地安装的包版本号
npm ls <packageName> # 本地包
npm ls <packageName> -g # 全局安装包

生产环境依赖包存放在 dependencies，devDependencies 储存开发环境中使用的依赖包。前者存放运行中使用的项目插件 ElementUI、Echarts 等。后者存放程序运行时并不需要的 gulp、webpack 等用来压缩，打包代码的工具。

package(-lock).json

通常不会将 node-modules 文件夹提交 Git，因通过 npm install 安装包时，可能出现版本不兼容、补丁或次版本缺陷等问题。而这也是 package.json 的特殊弊端。在 NPM5 引入的 package-lock.json 文件可以跟踪并固化被安装的每个软件包的确切版本，以便包模块以相同的方式被 100％ 复制。

当 npm 包某些功能布新的版本，通常的做法是直接去修改 package.json 到指定版本。package-lock.json 文件中 integrity 用于校验软件包；resolved 指向软件包的位置。

在 package.json 中，可以使用 semver 表示法设置要升级到的版本。版本号严格按照 "主版本号.次版本号.修订号" 格式命名。当某个版本改动比较大、并非稳定而且可能无法满足预期的兼容性需求时，会先发布一个先行版本。先行版本号可加在版本号的后边，即先加上一个连接号再加上一连串以句点分隔的标识符和版本编译信息。常见先行版本应说明内部版本 alpha、公测版本 beta、正式版的候选 rc Release candiate。主版本号一般作为有重大 api 改变，往往不向下兼容，次版本号也为小版本号，常新增功能，也向下兼容，修订号为补丁号，通常修复一些 bug。npm 安装包时允许使用一些特殊符号表示安装的版本范围。

// 先行版本号
a.b.c-beta.d
// 取最新的c的版本号值,a与b保持不变
~a.b.c
// 取b和c均为最新版本号,a保持不变
^a.b.c
// 始终使用确切的版本
a.b.c

如果自行发布包，在修改模块后需要更新发布版本，可以通过 Semver 规范的命令来完成，避免版本混乱。

npm version patch => 升级修订版本号
npm version minor => 升级次版本号
npm version major => 升级主版本号

npmrc

.npmrc 文件是 NPM 的用户配置文件。NPM 从命令行、环境变量和 npmrc 文件中获取其配置设置。使用 npm config 相关命令可用于更新和编辑用户和全局 npmrc 文件的内容。

# 查看路径
npm config get userconfig
# 显示当前仓库源
npm config get registry
# 设置国内镜像|官方镜像
npm config set registry=https://registry.npm.taobao.org|https://registry.npmjs.org

yarn

yarn 相比 npm 会缓存下载的每个包，且并行操作提高资源利用率的行为，会比同类包管理工具更快。

# 通过 npm 安装
npm install --global yarn
# Check installation
yarn --version
# 初始化
yarn init
# 添加依赖
yarn add [package]@[version]|[tag] --dev|--peer|--optional
# 升级依赖
yarn upgrade [package]@[version]|[tag]
# 删除依赖
yarn remove [package]
# 安装所有依赖
yarn | yarn install
