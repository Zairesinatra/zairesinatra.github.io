回流(Reflow)与重绘 (Repaint)

I'm not in danger,I am the danger. —— 《Breaking Bad》

Ziyi Xie

Jun 1, 2020 • 4 min read

前言

浏览器解析渲染机制作为前端开发是必须掌握的。更多详情zs推荐查询此网。

先上结论：回流必引起重绘，重绘不一定回流。

正文

浏览器采用 Flow Based Layout 流式布局模型，也就是常说的文档流。

浏览器渲染页面产生 RenderTree 前需要先构建 DOM 和 CSSOM 树。HTML 标记转换成文档对象模型 (DOM)；CSS 样式转换成 CSS 对象模型 (CSSOM)。

通过 RenderTree 可以将所有节点的样式计算成在页面上的大小和位置，并通过遍历节点绘成页面。

回流 (Reflow)

回流是因渲染树中元素的 尺寸、结构、属性 发生改变，使浏览器重新渲染部分或全部文档。

  • 浏览器视窗改变与页面首次渲染
  • 元素内容（文字数量或图片大小等等）、尺寸、字体或位置发生改变
  • 添加或者删除可见的DOM元素
  • 激活 CSS 伪类（例如：:hover）
  • 查询属性或调用计算方法（引擎会重新渲染来确保获取的值是实时的）

重绘 (Repaint)

当页面中元素样式的改变而不影响它在文档流中的位置时，浏览器会将新样式赋予给元素并重新绘制它，这个过程称为重绘。

  • visibility:hidden; (仅隐藏内容，并未脱离文档流，继续占据页面的空间)
  • color、background-color:...; (仅颜色改变)

优化与避免：

优化：
现代浏览器将可能引起回流和重绘的操作放入特定队列，进行批量处理。当任务数量或者时间间隔达到一个阈值的，浏览器就会将队列清空，进行一次批处理，这样可以把多次回流和重绘变成一次。例外是需要确定精确值的操作：clientWidth|Height|Top|Left(获取元素的高、宽度，包括填充)、offsetWidth|Height|Top|Left(距离父盒子偏移，不算入border)、scrollWidth|Height|Top|Left(以像素为单位返回元素的整个宽度，包括填充，但不包括边框、滚动条或边距)、getComputedStyle()(获取指定元素的所有实际（计算）CSS 属性和值)、getBoundingClientRect()(返回元素的大小及其相对于视口的位置)...

避免：

CSS

  • 避免网站整体使用的老旧的 table 布局（表格数据无伤大雅）。
  • 避免设置多层内联样式。（根据 css 权重级的特性，内联样式只能被 !important 所覆盖）
  • 避免使用CSS计算表达式（如：calc()）。

JavaScript

  • 避免频繁操作样式，最好一次性重写 style 属性，或者将样式列表定义为 class 并一次性更改class属性。
  • 避免频繁操作DOM，尽可能的 DOM 操作通过开辟 DocumentFragment 解决，最后再添加到文档中。
<ul id="list"></ul>

var list = document.querySelector('#list')
var fruits = ['Apple', 'Orange', 'Banana', 'Melon']

var fragment = new DocumentFragment()

fruits.forEach(function (fruit) {
  var li = document.createElement('li')
  li.innerHTML = fruit
  fragment.appendChild(li)
})

list.appendChild(fragment)
// Apple
// Orange
// Banana
// Melon

结束

简单来说：

  1. 会引起元素或周围的元素位置变化则 reflow，如窗口、字体大小、元素位置改变...；不会引起位置变化，仅存在变色效果则 repaint。
  2. 回流性能消耗较重绘要高，因回流的影响也会冒泡作用于周边及父元素。
  3. display:none;因不占据物理空间，因此涉及了dom树的操作，所以会引起重绘和回流；visibility:hiden; 元素虽然不可见但是却占据物理空间，因此不影响dom树的改变，所以只会引起回流。
