Null与Undefined区别

“以小明大，见一叶落而知岁之将暮，睹瓶中之冰，而知天下之寒。”——《淮南子·说山川》

Ziyi Xie

Apr 22, 2020 • 3 min read

通俗来说null与undefined概念较简单,但是这里有意思的是在基本数据类型中JavaScript语言存在两个表示"无"的值，这里拎出来比较一二。

控制台test如下：

if (!undefined) 
    console.log('undefined is false');
// undefined is false

if (!null) 
    console.log('null is false');
// null is false

undefined == null
// 抽象（非严格）相等比较 (==) true

undefined === null
// 严格相等比较 (===) false

null是一个表示"无"的对象，转为数值时为0。
undefined是一个表示"无"的原始值，转为数值时为NaN。
同时undefined不是一个有效的JSON，而null是。

定义：

undefined是全局对象的一个属性。也就是说，它是全局作用域的一个变量。undefined的最初值就是原始数据类型undefined。如果方法或者是语句中操作的变量没有被赋值，则会返回undefined。

值 null 特指对象的值未设置。值 null 是一个字面量，不像 undefined，它不是全局对象的一个属性。null 是表示缺少的标识，指示变量未指向任何对象。把 null 作为尚未创建的对象，也许更好理解。在 API 中，null 常在返回类型应是一个对象，但没有关联的值的地方使用。

常用方式与表述：

null：

  • 对象原型链的终点。
  • 作为函数的参数，表示该函数的参数不是对象。

undefined：

  • 调用函数时，应该提供的参数没有提供，该参数等于undefined。
  • 函数没有返回值时，默认返回undefined。
**上述代码如下：**
//null->对象原型链的终点
Object.prototype.__proto__ === null

//undefined->调用函数时，应该提供的参数没有提供，该参数等于undefined
var a="zs"
function hrb(){
    console.log(a)
    a="xc"
}
hrb() // undefined
