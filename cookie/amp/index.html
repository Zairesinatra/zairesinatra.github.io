Cookie, LocalStorage 与 SessionStorage

Ziyi Xie - 12 Apr 2020

前言

每次聊 HTML5 都能扯到 Cookie、localStorage、SessionStorage。这篇文章旨在对比阐述这三者关系，具体 Web Storage API 的使用可以参考MDN的文档。想看的话点这里吧。

正文

  • Cookie

Cookie 中文翻译即“曲奇🍪”。顾名思义，小是主要的特点，其大小通常限制为 4KB 左右，是 Netscape 公司最先提出。通常作用于保存登录信息，比如在初次页面登录后 chrome 会提醒用户是否储存账户信息，这通常就是在 Cookie 中存入一段辨别用户身份的数据来实现的。

如下图，通常是由web服务器存储在用户机器硬盘上的文本文件。这个文本文件里存着用户的用户名、密码、浏览网页、停滞事件等...

每一次客户发送http请求，浏览器都会查看是否存在对应的cookie。有则自动添加在 RequestHeader中cookie字段。

---

  • LocalStorage

LocalStorage 是 H5 标准中的新增，取代了 globalStorage 。但早在 IE6 时代，就采取了userData 用于本地存储，而当时考虑到浏览器兼容性，更通用的方案是大家都是用过的 Flash。而如今，localStorage 被更多浏览器所支持。但 LocalStorage 最重要的是持久化的本地存储。除非通过 Javascript 代码或者清除缓存，否则数据一直存在。

  • SessionStorage

SessionStorage 直译过来是“会话存储”。SessionStorage 可以将一部分数据在当前会话页面中保存下来，刷新页面数据依旧存在。但当页面关闭后，sessionStorage 中的数据就会被清空。因保存数据的生命周期较 localStorage 更短，所以常应用来保存 Token 。

// 保存数据到 sessionStorage
sessionStorage.setItem('key', 'value');

// 从 sessionStorage 获取数据
let data = sessionStorage.getItem('key');

// 从 sessionStorage 删除保存的数据
sessionStorage.removeItem('key');

// 从 sessionStorage 删除所有保存的数据
sessionStorage.clear();
  • 三者对比
特性 Cookie LocalStorage SessionStorage
数据的生命期 由服务器生成，可设置失效时间Expires 除非被清除，否则永久保存 当前会话下有效，关闭页面后被清除
存放数据大小 4K左右 一般为5MB
与服务器端通信 携带在HTTP请求头 仅在浏览器中保存，不参与和服务器的通信
  • 封装 Cookie 的设置、读取与删除
// 设置Cookie
function setCookie (key, value, t) { // t设置cookie多久过期,通常设置天数
  var myTime = new Date();
  myTime.setDate(myTime.getDate() + t);
  document.cookie = key + "=" + value + ";expires=" + myTime.toUTCString();
}

// 读取Cookie
function getCookie (name) {
  var arr = document.cookie.split(';'); // 通过";"分隔cookie
  for (var i = 0 ; i < arr.length ; i++) {
    var arr1 = arr[i].split('=');
    if (arr1[0] == name) {
      return arr1[1];
    }
  }
  return ""; // 遍历完成没有找到则返回空
}

// 删除Cookie
function removeCookie (key) {
  setCookie(key, "1", -1) // 设置的value随意,主要是t的意思则是已过期,可删除
}
