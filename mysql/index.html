MySQL数据库的基本点在这里总结一下。毕竟宏观项目中对于数据操作也是必不可少的。

正文

MySQL 概念：

不论传统数据库: MySQL、Oracle、SQL Server 还是新型数据库: Mongodb ，都是通过增、删、改、查等系列操作来进行组织、存储、管理数据。而数据库结构又通常分为数据库（database）、数据表（table）、数据行（row）、字段（field）四种组成。

MySQL 安装与卸载：

  • 安装：

MySQL Server: 提供数据储存、数据服务
MySQL Workbench: 可视化MySQL管理工具-方便操作存储的数据

  • 卸载：
sudo rm /usr/local/mysql
sudo rm -rf /usr/local/mysql*
sudo rm -rf /Library/StartupItems/MySQLCOM
sudo rm -rf /Library/Receipts/mysql*

MySQL 的常用路径：

MacOS 可通过在系统偏好设置中 mysql 项目的 configuration 栏查询到 mysql 具体各内容的安装位置。
而往往都会存储在 /usr/local/mysql 这个位置 => open /usr/local/mysql。

MySQL 操作

  • 配置环境变量：
vi ~/.zshrc

# mysql
export PATH="/usr/local/mysql/bin:$PATH"
alias  mysql='/usr/local/mysql/bin/mysql'

# 更新配置
source ~/.zshrc
  • 数据库、数据表：

通过工具栏中的类🛢形创建的数据库注意命名规范 => 不能中文和空格

特殊含义:
PK(primary key):主键唯一标识、NN(not null):值不允许为空、UQ(Unique):值唯一、AI(Auto Increment):值自动增长

  • 导入.sql文件：

预备操作：创建需以导入的.sql文件名命名的新数据库。（此处以 mydb 为例）

terminal 输入：zsh mysql -u root -p

出现的 mysql 中输入： mysql>use mydb(使用那个数据库的名称)

导入文件命令：zsh mysql>source /Users/whoami/Desktop/.../db/mydb.sql

完成导入如图：

测试导入的数据库文件：zsh open /usr/local/mysql/data

此时可能会因权限问题出现以下报错：

解决：zsh sudo chmod -R 777 /usr/local/mysql/data/

退出mysql：zsh exit;

BUG 解决：

/Users/.../node_modules/mysql/lib/protocol/Parser.js:80
        throw err; // Rethrow non-MySQL errors
        ^

Error: ER_ACCESS_DENIED_ERROR: Access denied for user 'root'@'localhost' (using password: YES)

这里报错的原因是配置的数据库密码与设置的密码不一致，在配置数据库文件中找到对应的配置修改default.json：

Your connection attempt failed for user 'root' to the MySQL server at localhost:3306:
Unable to connect to localhost:3306

Can't connect to local MySQL server through socket '/tmp/mysql.sock' (2)

mysql.sock 这个文件是用于 socket 连接的文件，也就是只有你的守护进程启动起来这个文件才存在。 那么，请重启 MySQL。

常用命令：

#  查看 MySQL 是否启动，若没有启动就执行
ps -ef |grep mysql
mysql start

# 登陆 MySQL 需执行
mysql -uroot

# 启动 MySQL 服务
sudo /usr/local/MySQL/support-files/mysql.server start

# 停止 MySQL 服务
sudo /usr/local/mysql/support-files/mysql.server stop

# 重启 MySQL 服务
sudo /usr/local/mysql/support-files/mysql.server restart

以上

总结：

项目中可能会涉及操作 MySQL 的辅助软件，如 windows 可以采用 phpstudy 快速导入 .sql 文件。而在 MacOS 中少有同类产品（嗯，我找过了，放弃抵抗）。

那么在 terminal 使用 shell 命令完成操作就会比盲目寻找捷径更为捷径。
